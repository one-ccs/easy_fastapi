# 0.x → 1.0 迁移指南

1.0 是大重构，**不保留向后兼容层**。建议新项目直接用 1.0；旧项目按以下步骤逐项迁移。

## 1. 概述

| 维度 | 0.x | 1.0 |
|---|---|---|
| 架构 | 单体运行时 + 手动接线 | core/ext 运行时 + CLI/scaffold/generator 生成层 |
| 配置 | `Config()` 全局单例，`extra='allow'` | `ConfigLoader.from_yaml()`，`extra='forbid'`，env overlay |
| ORM | Tortoise 硬编码 | 三选一（tortoise/sqlalchemy/sqlmodel），统一协议 |
| 认证 | `EasyAuthentication` + 旧 `authentication/` | `AuthExtension`，跨 ORM，pwdlib + PyJWT |
| 前端 | 无脚手架前端 | Vue3 全家桶 + OpenAPI SDK（可选） |
| CLI | `manager.py` + 旧命令 | `efa create/run/db/gen` |
| 包管理 | pip | uv |

## 2. 配置迁移

### 2.1 Config 单例 → ConfigLoader

**旧（0.x）：**
```python
from easy_fastapi import Config
config = Config()
db_url = config.database.url
```

**新（1.0）：**
```yaml
# easy-fastapi.yaml
fastapi:
  root_path: /api
database:
  dialect: sqlite
  database: db.sqlite3
```

```python
from easy_fastapi import ConfigLoader
loader = ConfigLoader.from_yaml(Path("easy-fastapi.yaml"))
from easy_fastapi.core.config.models import FastAPIConfig
cfg = loader.section("fastapi", FastAPIConfig)
```

### 2.2 extra='allow' → extra='forbid'

旧版多余 YAML 键静默忽略。1.0 报 `ConfigError`。检查 YAML，删除不再使用的键。

### 2.3 环境变量覆盖

新增规则：`EFA_<SECTION>__<FIELD>` 覆盖 YAML 值。

扩展配置在 `easy_fastapi` 段下，env 前缀需包含完整路径：

```bash
# fastapi 段（顶层，不变）
EFA_FASTAPI__ROOT_PATH=/api

# 扩展配置（嵌套在 easy_fastapi 段下，需加 EASY_FASTAPI__ 前缀）
EFA_EASY_FASTAPI__AUTH__SECRET=prod-key
EFA_EASY_FASTAPI__DATABASE__DIALECT=sqlite
EFA_EASY_FASTAPI__REDIS__URL=redis://prod:6379/0
```

## 3. 代码迁移

### 3.1 EasyFastAPI 装配

**旧：**
```python
from easy_fastapi import EasyFastAPI
easy = EasyFastAPI(app)
easy.init_app()  # 旧方法
```

**新：**
```python
from easy_fastapi import EasyFastAPI
from easy_fastapi.ext.orm.tortoise.extension import TortoiseExtension
from easy_fastapi.ext.auth.extension import AuthExtension

easy = EasyFastAPI(app, config_path="easy-fastapi.yaml")
easy.use(TortoiseExtension())
easy.use(AuthExtension())
```

### 3.2 旧 import → 新路径

| 旧 import | 新路径 |
|---|---|
| `from easy_fastapi import Config` | `from easy_fastapi import ConfigLoader` |
| `from easy_fastapi import init_tortoise` | `easy.use(TortoiseExtension())` |
| `from easy_fastapi import authentication` | `from easy_fastapi.ext.auth.extension import AuthExtension` |
| `from easy_fastapi import persistence` | `ctx.require("persistence", Persistence)` |
| `from easy_fastapi import Result, JSONResult` | `from easy_fastapi import BaseResult, Result, ResponseResult` |

### 3.3 认证

`EasyAuthentication` → `AuthExtension`。配置从 `easy_fastapi.authentication` section 移至 `auth` section：

```yaml
# 旧
easy_fastapi:
  authentication:
    secret_key: xxx

# 新
auth:
  secret: xxx
```

### 3.4 持久化选择器

旧 `Persistence()` 选择器 → `ctx.require("persistence", Persistence)`。Redis override 经 `RedisExtension` 自动完成。

## 4. 删除清单

| 已删模块 | 替代 |
|---|---|
| `easy_fastapi/config.py` | `easy_fastapi/core/config/` |
| `easy_fastapi/management.py` | `easy_fastapi/cli/commands/db.py` |
| `easy_fastapi/db.py` | `easy_fastapi/ext/orm/` + `easy_fastapi/cli/commands/db.py` |
| `easy_fastapi/generator.py` | `easy_fastapi/generator/codegen.py` |
| `easy_fastapi/result.py` | `easy_fastapi.core.result`（`BaseResult`/`Result`/`ResponseResult` 顶层导出已恢复） |
| `easy_fastapi/persistence.py` | `easy_fastapi/core/persistence/` + `easy_fastapi/ext/redis/` |
| `easy_fastapi/authentication/` | `easy_fastapi/ext/auth/` |
| `easy_fastapi/exception/` | `easy_fastapi/core/exceptions.py` |

## 5. 数据库迁移

- Tortoise：`efa db init`（非幂等）→ `efa db migrate` → `efa db upgrade`
- SQLAlchemy / SQLModel：Alembic，`efa db init` → `efa db migrate` → `efa db upgrade`
- 开发用：`efa db sync`（直接建表，跳过迁移）

## 6. 前端

0.x 无脚手架前端。1.0 可选前端最小骨架：

```bash
efa create myapp --frontend --database --orm tortoise --db-dialect mysql --auth
```

生成 `frontend/` 最小 pnpm monorepo 骨架（`packages/api-sdk` + `apps/` 占位），不预置具体前端应用。

## 7. 包管理

pip → uv：

```bash
# 旧
pip install easy_fastapi
pip install tortoise-orm

# 新
uv add easy_fastapi
uv add tortoise-orm
```

`efa` CLI 安装：

```bash
uv tool install easy-fastapi-cli
```
